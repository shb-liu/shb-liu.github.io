Github Pages自定义域名开启HTTPS

发表于 2020-04-06 分类于 Github Pages 阅读次数：

环境

  • Github Pages
  • 自定义域名

问题

为Github Pages添加自定义域名后无法开启HTTPS支持。

解决方案

将自定义域名绑定到Github Pages后，在博客Repository的Settings中，Enforce HTTPS选项无法勾选。

再次查阅Github Pages相关帮助后，发现应该在域名的DNS解析处更改A Record到指定IP。具体如下：

  1. 在DNS服务供应商处创建4个A record（或ALIAS，ANAME三者之一）分别指向以下IP地址：

  2. 删除DNS设置内其他指向Github的A record。

  3. 进入博客Repository的Settings中，在Custom domain中填入自定义域名。

  4. 等待一段时间后，再次进入Settings查看Enforce HTTPS是否开启，如果没有开启手动点选。

其他

在设置后，可能遇到Settings中Custom domain相关内容变动的情况。建议多等待一段时间，并使用网络工具查看自定义域名所对应的IP是否在添加的4个IP中。如果没有问题，还是无法打开博客，可以再等待一段时间。或使用浏览器的无痕浏览进行连接。
