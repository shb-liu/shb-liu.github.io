MacOS扩展屏分辨率无法正常识别

发表于 2020-04-05 分类于 MacOS 阅读次数：

环境

  • MacOS 10.15.4
  • MAYA显示器X212e

问题

Macbook使用miniDP转VGA时，扩展屏无法选择正确的分辨率。

解决方案

朋友所在的公司淘汰了一批显示器，于是便收了一台MAYA X212e显示器，作为分屏使用。该显示器最佳分辨率为1600 * 900，接口仅有VGA。在搜索转接线时发现原来Macbook的雷电接口可以和miniDP接口通用，因此决定使用miniDP转单VGA的转接线。某东上随便搜了一圈，发现各转接线均使用的是RTD2166芯片解决方案，于是选择了一个销量大，价格便宜的品牌。

到手之后发现分辨率仅有两个4:3的选项，而不能选择显示器的最佳分辨率（1600 * 900）。在网上搜索解决方案时，大多给的解决方案和官网的解决方案一样：

  1. 在 Mac 上，选取苹果菜单>“系统偏好设置”，点按“显示器”，然后点按“显示器”。
  2. 按住 Option 键点按“缩放”以查看第二台显示器的其他分辨率。
  3. 选择第二台显示器的缩放分辨率选项。

按照方法操作后，仅出现了1920 * 1080的分辨率，并没有1600 * 900的选项。本以为只能这样使用4:3的分线率来使用分屏了。

不过在搜索中发现Windows中有时也会出现该问题，可以通过使用自定义分辨率来解决。出现这种问题的原因是电脑硬件或系统版本的原因导致无法识别显示器自带的提供型号及最佳分辨率的设置。于是猜测是转接头的芯片不支持识别1600 * 900的分辨率。

原转接头退货，查看其他转接头参数，发现miniDP转VGA+HDMI的芯片与之前的不同，分别是ANX9833和PS8339B。于是下单。收货后插上之后果然可以识别出扩展平的最佳分辨率1600 * 900。完美解决。
